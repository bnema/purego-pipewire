# purego-pipewire

Pure Go bindings for PipeWire using [purego](https://github.com/ebitengine/purego).

## Architecture

This package follows the same layer split as other purego-* projects:

- `internal/capi` — generated raw function types and registration helpers
- `internal/ports/{in,out}` — generated port interfaces
- `internal/core` — handwritten lifecycle logic
- `internal/loader` — `dlopen` for PipeWire and SPA libs
- `pipewire/` — public Go-facing wrappers
- `spa/` — public constants and helpers

## Regenerating bindings

Normal contributors should run:

    go generate ./...

This uses checked-in model files under `gen/` (e.g. `gen/pipewire.json`).

Mocks are generated by [mockery](https://github.com/vektra/mockery):

    mockery

### Maintainer workflow

Maintainers can refresh the model from local PipeWire and SPA headers with:

    ./scripts/sync-headers.sh
    go generate ./...

The sync script is currently a placeholder for the planned header-import workflow.

## Requirements

- Go 1.26.1+
- Linux with PipeWire 0.3 runtime libraries

## Integration tests

Run integration tests (requires PipeWire runtime):

    go test -tags integration ./integration ./pipewire -v
